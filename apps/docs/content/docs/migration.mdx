---
title: Migration from lunacord.js 0.x
description: Changes between the original single-package Lunacord and the new @lunacord/* monorepo.
---

## Install

Before:

```bash
bun add lunacord.js
bun add redis # optional
```

After (scoped packages — best when you want minimal transitive deps):

```bash
bun add @lunacord/core @lunacord/discordjs discord.js
# Optional
bun add @lunacord/plugins @lunacord/lyrics @lunacord/cache-redis redis
```

Alternatively, **`lunacord.js`** remains the unscoped umbrella: it re-exports `@lunacord/core`, `@lunacord/discordjs`, builtins from `@lunacord/plugins`, `@lunacord/lyrics`, and `@lunacord/cache-redis`, and ships **`createLunacordMusicClient()`** to build a discord.js `Client` + `MusicKit` in one step. Subpaths `lunacord.js/plugins`, `lunacord.js/lyrics`, and `lunacord.js/cache` match the old layout.

## Imports

| Before | After (scoped) | Still valid on `lunacord.js` |
|--------|----------------|-----------------------------|
| `import { Lunacord } from "lunacord.js"` | `import { Lunacord } from "@lunacord/core"` | root export |
| `import { PluginBuilder, createLoggerPlugin } from "lunacord.js/plugins"` | `import { PluginBuilder, createLoggerPlugin } from "@lunacord/plugins"` | `./plugins` subpath |
| `import { LyricsClient } from "lunacord.js/lyrics"` | `import { LyricsClient } from "@lunacord/lyrics"` | `./lyrics` subpath |
| `import { RedisCacheStore } from "lunacord.js/cache"` | `import { RedisCacheStore } from "@lunacord/cache-redis"` | `./cache` subpath |
| `import { LavalinkConnectionError } from "lunacord.js/errors"` | `import { LavalinkConnectionError } from "@lunacord/core"` | root export |

`SearchProvider`, schemas, and cache primitives are now re-exported from `@lunacord/core` root.

## Construction

```diff
- const lunacord = new Lunacord({
-   userId, numShards, nodes: [{ ... }],
-   sendGatewayPayload: (guildId, payload) => guild.shard.send(payload),
-   lyrics: { genius: { ... } },
- });

+ const lunacord = Lunacord.create()
+   .userId(userId)
+   .shards(numShards)
+   .node({ id: "main", host, port, password })
+   .build();
+
+ lunacord.lyrics(
+   LyricsClient.create().provider.genius({ ... }).provider.lyricsOvh().build()
+ );
```

## Identity binding is optional

`userId` and `numShards` are no longer required at construction. Adapters (`MusicKit`) bind them after the Discord client becomes ready via `lunacord.bindIdentity({ userId, numShards })`.

## Unified debug event

The node-scoped `debug` event is now folded into a single `debug` firehose on `Lunacord`:

```diff
- lunacord.on("debug", (event) => console.log(event.node.id, event.category, event.message));
+ lunacord.on("debug", (event) => console.log(event.nodeId, event.scope, event.message));
```

## Plugin API bumped to v2

Plugins declaring `apiVersion: "1"` still load with a deprecation warning. New plugins should target `apiVersion: "2"` (which is `LUNACORD_PLUGIN_API_VERSION`). The only new hook is `onPlayerRestore` for persistence-aware behavior.

## Redis cache moved out of core

`RedisCacheStore` lives in `@lunacord/cache-redis`. A new `RedisPersistenceAdapter` in the same package provides post-restart rehydration.

## `lyrics` option now takes a provider

`options.lyrics` used to be `LyricsOptions` (Genius config). It is now a `LyricsProvider` instance (from `@lunacord/lyrics`). This removes the hard dependency on Genius/LyricsOvh from core and makes lyrics fully pluggable.

## Batteries-included MusicKit

Nothing in the old API is removed — but if you were building a bot from scratch, consider using `@lunacord/discordjs` instead of wiring discord.js by hand. It absorbs ~100 lines of adapter boilerplate.
